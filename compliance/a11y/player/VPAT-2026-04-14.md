## Findings (by WCAG criterion)

### 1.3.1 Info and Relationships (Level A)

**Conformance Level:** Does Not Support

**Remarks:**

- On `/player`, "Powered by Remotion" and "Reference application" are visually headings but marked up with `<strong>`. Use proper heading elements.
- On `/player`, the input field under "Demo" has no persistent `<label>` — only a placeholder.

### 2.1.1 Keyboard (Level A)

**Conformance Level:** Does Not Support

**Remarks:**

- On `/player`, "Demo" color-selection controls cannot be operated with the keyboard.

### 2.4.4 Link Purpose (In Context) (Level A)

**Conformance Level:** Does Not Support

**Remarks:**

- On `/player`, multiple links share identical text ("Learn more", "Read docs") but lead to different destinations.

### 2.4.6 Headings and Labels (Level AA)

**Conformance Level:** Partially Supports

**Remarks:**

- See 1.3.1 — visually styled headings not marked up as heading elements; placeholder-only input label.

### 3.3.2 Labels or Instructions (Level A)

**Conformance Level:** Does Not Support

**Remarks:**

- `/player` "Demo" input relies on placeholder only — no persistent visible label.

### 4.1.2 Name, Role, Value (Level A)

**Conformance Level:** Does Not Support

**Remarks:**

- `/player` "Demo" color-selection interactive elements have no accessible role or name.

### 4.1.3 Status Messages (Level AA)

**Conformance Level:** Does Not Support

**Remarks:**

- On `/player`, when a color is selected in the "Demo", the information in the nearby video changes without being announced to screen readers. Use `aria-live="polite"` on the updated region.

### Overall

The Player marketing/demo page fails multiple WCAG 2.1 Level A and AA criteria in the sampled scope. No findings were recorded for `/docs/player/buffer-state`. Findings are derived from the remotion.dev RGAA 4.1.2 audit (Victoria, 14.04.2026). A full VPAT would require broader sampling of the Player library in context (e.g. host apps using the `<Player>` component).
