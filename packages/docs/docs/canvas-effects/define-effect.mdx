# defineEffect()<AvailableFrom v="4.0.452" />

Helper for declaring a [canvas effect](/docs/canvas-effects). Wrapping a definition in `defineEffect()` does not change its runtime behavior — it only enables type inference for the parameter and state generics.

```tsx twoslash title="src/effects/identity.ts"
import {defineEffect, createDescriptor} from 'remotion';

const identityDef = defineEffect<{}, null>({
	type: 'remotion-docs/identity',
	backend: '2d',
	setup: () => null,
	apply: ({source, target, width, height}) => {
		const ctx = target.getContext('2d');
		if (!ctx) {
			return;
		}

		ctx.clearRect(0, 0, width, height);
		ctx.drawImage(source, 0, 0, width, height);
	},
});

export const identity = () => createDescriptor(identityDef, {});
```

## Definition fields

### type

A unique string identifier. Used in error messages and dev-tools; not used for caching.

### backend

One of `'2d' | 'webgl2' | 'webgpu'`. Determines which kind of context the runtime hands you in `target.getContext(...)`.

### setup

`(target: HTMLCanvasElement) => S | Promise<S>`

Called once per effect identity, per chain. Use it to compile shaders, allocate textures, or upload static buffers. The returned value is passed back to `apply()` as `state`.

The runtime caches the returned state in a `WeakMap<EffectDefinition, ...>`, so re-rendering does not re-run setup. The cache is invalidated only when the chain unmounts or the canvas dimensions change.

### apply

`(params: EffectApplyParams<P, S>) => void | Promise<void>`

Called every frame. Receives:

#### source

A `CanvasImageSource` containing the input image (the output of the previous effect, or the source component's image for the first effect in the chain).

#### target

The `HTMLCanvasElement` to draw the output into. The runtime owns this canvas; the effect must not hold a reference to it across frames.

#### state

The value returned from `setup()`.

#### params

The parameters passed in the [`createDescriptor()`](#createdescriptor) call.

#### frame

The current frame number, equivalent to [`useCurrentFrame()`](/docs/use-current-frame).

#### width / height / pixelRatio

The chain's dimensions and the host's pixel ratio.

#### gpuDevice

A `GPUDevice` if any effect in the chain uses the `'webgpu'` backend, otherwise `null`. The type is `unknown` to avoid pulling `@webgpu/types` into core; cast it at the use site.

### cleanup?

`(state: S) => void`

Called when the chain unmounts. Use it to free GPU resources allocated in `setup()`. Optional.

## createDescriptor()

Pairs a definition with a set of parameters to produce an `EffectDescriptor` that can be passed to a source component's `effects` prop.

```ts twoslash
import {defineEffect, createDescriptor} from 'remotion';

const identityDef = defineEffect<{}, null>({
	type: 'remotion-docs/identity',
	backend: '2d',
	setup: () => null,
	apply: () => undefined,
});
// ---cut---
const descriptor = createDescriptor(identityDef, {});
```

Effect authors typically expose a thin wrapper:

```ts twoslash
import {defineEffect, createDescriptor} from 'remotion';

type WaveParams = {amplitude: number};
const waveDef = defineEffect<WaveParams, null>({
	type: 'remotion-docs/wave',
	backend: '2d',
	setup: () => null,
	apply: () => undefined,
});
// ---cut---
export const wave = (params: WaveParams) => createDescriptor(waveDef, params);
```
