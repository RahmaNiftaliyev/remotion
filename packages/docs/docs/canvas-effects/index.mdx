# Canvas effects<AvailableFrom v="4.0.452" />

Canvas effects let you build After Effectsâ€“style per-pixel transformations directly on a `<canvas>`, composing 2D, WebGL2, and (in the future) WebGPU passes inside a single chain.

The system has two layers:

- The runtime, source components ([`<Solid>`](/docs/solid), [`<Experimental.HtmlInCanvas>`](/docs/canvas-effects/html-in-canvas)), and the [`defineEffect()`](/docs/canvas-effects/define-effect) helper ship inside `remotion`.
- Sample effects ([`wave`](/docs/canvas-effects/wave), [`tint`](/docs/canvas-effects/tint), [`blur`](/docs/canvas-effects/blur)) ship inside [`@remotion/canvas-effects`](/docs/canvas-effects).

## Concepts

### Source components

A source component owns a chain. It produces a `CanvasImageSource` (the input image) for every frame and renders the chain output into a visible `<canvas>`. The first source components are:

- [`<Solid>`](/docs/solid) â€” a flat-color source.
- [`<Experimental.HtmlInCanvas>`](/docs/canvas-effects/html-in-canvas) â€” rasterizes DOM children into a canvas via the [WICG `html-in-canvas` proposal](https://github.com/WICG/html-in-canvas).

### Effects

An effect is a description of how to transform an input image into an output image. Each effect targets a single backend (`'2d'`, `'webgl2'`, or `'webgpu'`) and defines `setup`, `apply`, and (optionally) `cleanup` lifecycle hooks. Authors declare effects with [`defineEffect()`](/docs/canvas-effects/define-effect).

Effects are passed into a source component as an array via the `effects` prop:

```tsx twoslash title="src/Demo.tsx"
import {blur, tint} from '@remotion/canvas-effects';
import {Solid} from 'remotion';

export const Demo = () => (
	<Solid
		color="hotpink"
		width={1920}
		height={1080}
		effects={[blur({radius: 24}), tint({color: 'cyan', amount: 0.4})]}
	/>
);
```

### Chain semantics

The chain runtime processes effects sequentially. Adjacent effects sharing the same backend are grouped into a single "run" and ping-ponged between two scratch canvases. Between runs, the runtime transfers the intermediate image via `createImageBitmap()` so the next backend can sample it.

The runtime owns one `delayRender()`/`continueRender()` pair per frame and per chain.

### Alpha and color space

All scratch canvases use **premultiplied alpha** and **sRGB** storage. Effects that perform color math in linear space must convert to/from sRGB themselves. Source components write their input image in this format; the runtime preserves it through every transition.
