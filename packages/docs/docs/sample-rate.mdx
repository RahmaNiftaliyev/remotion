The audio sample rate determines how many audio samples per second are stored in the output file. A higher sample rate preserves more detail in high-frequency content.

## Default behavior

By default, Remotion renders all audio at **48000 Hz** (48 kHz). This is a broadcast standard and produces good results for most content.

All audio sources — regardless of their original sample rate — are resampled to the output sample rate during rendering. This means a 44100 Hz source embedded in a Remotion composition will be upsampled to 48000 Hz.

## Resampling artifacts

Resampling is a lossy transformation. If your source audio has a different sample rate than the output, the conversion may introduce subtle quality degradation. This is most noticeable with:

- Music mastered at 44100 Hz (CD quality) being upsampled to 48000 Hz
- High-quality recordings at 96000 Hz being downsampled
- Audio containing sharp transients or high-frequency detail

To avoid these artifacts, match the output sample rate to the sample rate of your source audio.

## Overriding the sample rate<AvailableFrom v="4.0.448" />

You can change the output sample rate using one of three methods. They are listed in order of priority — the first one that is set takes precedence.

### Via CLI flag

```bash
npx remotion render MyComp --sample-rate=44100
```

### Via config file

```ts twoslash title="remotion.config.ts"
import {Config} from '@remotion/cli/config';

Config.setSampleRate(44100);
```

### Via `renderMedia()`

```ts twoslash title="render.ts"
// @module: es2022
// @target: es2017
import {bundle} from '@remotion/bundler';
import {renderMedia, selectComposition} from '@remotion/renderer';

const bundled = await bundle({entryPoint: './src/index.ts'});

const composition = await selectComposition({
  serveUrl: bundled,
  id: 'MyComp',
});

await renderMedia({
  composition,
  serveUrl: bundled,
  codec: 'h264',
  sampleRate: 44100,
});
```

### Via `renderMediaOnWeb()`

```ts twoslash title="web-render.ts"
// @module: es2022
// @target: es2017
import {renderMediaOnWeb} from '@remotion/web-renderer';
const composition = {
  id: 'MyComp',
  durationInFrames: 100,
  fps: 30,
  width: 1920,
  height: 1080,
  defaultProps: {},
};
// ---cut---
await renderMediaOnWeb({
  composition,
  sampleRate: 44100,
});
```

### Via the Studio render dialog

In the [Remotion Studio](/docs/studio) render dialog, you can select the sample rate from a dropdown in the **Audio** tab.

## Per-composition sample rate

You can set the sample rate per composition using [`calculateMetadata()`](/docs/calculate-metadata). This is useful when your composition is driven by a media file with a known sample rate.

```ts twoslash title="src/Root.tsx"
// @filename: MyComp.tsx
import React from 'react';

export type MyCompProps = {
  src: string;
};

export const MyComp: React.FC<MyCompProps> = ({src}) => {
  return <div>{src}</div>;
};

// @filename: get-media-metadata.ts
import {Input, ALL_FORMATS, UrlSource} from 'mediabunny';

export const getMediaMetadata = async (src: string) => {
  const input = new Input({
    formats: ALL_FORMATS,
    source: new UrlSource(src, {
      getRetryDelay: () => null,
    }),
  });

  const durationInSeconds = await input.computeDuration();
  const videoTrack = await input.getPrimaryVideoTrack();
  const audioTrack = await input.getPrimaryAudioTrack();

  const dimensions = videoTrack
    ? {
        width: videoTrack.displayWidth,
        height: videoTrack.displayHeight,
      }
    : null;
  const packetStats = await videoTrack?.computePacketStats(50);
  const fps = packetStats?.averagePacketRate ?? null;
  const sampleRate = audioTrack?.sampleRate ?? null;

  return {
    durationInSeconds,
    dimensions,
    fps,
    sampleRate,
  };
};

// @filename: Root.tsx
// ---cut---
import React from 'react';
import {CalculateMetadataFunction, Composition} from 'remotion';
import {MyComp, MyCompProps} from './MyComp';
import {getMediaMetadata} from './get-media-metadata';

const calculateMetadata: CalculateMetadataFunction<
  MyCompProps
> = async ({props}) => {
  const metadata = await getMediaMetadata(props.src);

  return {
    props,
    durationInFrames: Math.ceil(
      (metadata.durationInSeconds ?? 1) * 30,
    ),
    fps: 30,
    width: metadata.dimensions?.width ?? 1920,
    height: metadata.dimensions?.height ?? 1080,
    defaultSampleRate: metadata.sampleRate ?? undefined,
  };
};

export const Root: React.FC = () => {
  return (
    <Composition
      id="MyComp"
      component={MyComp}
      durationInFrames={1}
      fps={30}
      width={1920}
      height={1080}
      defaultProps={{src: 'https://remotion.media/video.mp4'}}
      calculateMetadata={calculateMetadata}
    />
  );
};
```

If the media file has an audio track, the render sample rate will match the source. No CLI flag or config is needed.

The `defaultSampleRate` returned by `calculateMetadata()` has lower priority than the CLI flag and config file. See [Priority](#priority) for the full order.

## Mixing audio with different sample rates

When a composition uses multiple audio sources with different sample rates, all of them are resampled to the single output sample rate. There is no way to preserve multiple sample rates in a single output file — this is a fundamental limitation of audio containers.

Choose the sample rate that matches the majority of your sources, or the highest sample rate among them to minimize quality loss.

## Priority

When multiple sources define a sample rate, the highest-priority value wins:

- <Step>1</Step> `--sample-rate` CLI flag / `sampleRate` option in `renderMedia()`
- <Step>2</Step> `Config.setSampleRate()` in the config file
- <Step>3</Step> `defaultSampleRate` returned by [`calculateMetadata()`](/docs/calculate-metadata)
- <Step>4</Step> Default: `48000`

## Sample rate during preview

During preview in the [Remotion Studio](/docs/studio), audio is always played back at 48000 Hz. This cannot currently be changed.

This means previewed audio may sound slightly different from the final render if you use a custom sample rate. The rendered output will use the correct sample rate.

## Common sample rates

| Sample Rate | Use Case |
| --- | --- |
| 8000 Hz | Telephone, low-quality voice |
| 16000 Hz | Wideband speech, voice assistants |
| 22050 Hz | AM radio quality |
| 44100 Hz | CD audio, music production |
| 48000 Hz | Broadcast video, Remotion default |
| 96000 Hz | High-resolution audio |
